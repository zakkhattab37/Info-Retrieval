Search Algorithms Documentation

BM25 (Best Match 25)

The gold standard probabilistic ranking function.

How It Works: Uses probabilistic model with term frequency saturation and document length normalization
Formula:
BM25(D,Q) = IDF(qi) (f(qi,D) (k1+1)) / (f(qi,D) + k1(1-b+b|D|/avgdl))
Best Case: General-purpose searches
Worst Case: Very short queries (1-2 words)
Speed: Fast (O(nm))
Recommended For: Most searches, production systems, varying document lengths
Tip: This is the default and most reliable algorithm. Use it unless you have specific requirements.
TF-IDF (Vector Space Model)

Classic information retrieval algorithm based on term importance.

How It Works: Weights terms by frequency in document and rarity across corpus
Formula:
TF-IDF(t,d) = TF(t,d) log(N/df(t))
Best Case: Balanced document collections
Worst Case: Extreme length variations
Speed: Fast (O(nm))
Recommended For: Academic research, baseline comparisons, similar-length documents
Note: Traditional and well-studied. Good for understanding IR fundamentals.
Vector Space (TF)

Similarity-based matching using term frequency vectors.

How It Works: Represents documents and queries as vectors, calculates cosine similarity
Formula:
cos() = (DQ) / (||D|| ||Q||)
Best Case: Finding similar documents
Worst Case: Large document collections
Speed: Medium (O(nm), vector ops)
Recommended For: Similarity searches, recommendation systems, small-medium collections
Warning: Can be slower with large document sets due to vector operations.
Log TF-IDF

TF-IDF with logarithmic dampening for better scaling.

How It Works: Applies logarithmic scaling to term frequencies to reduce impact of very frequent terms
Formula:
Log-TF-IDF(t,d) = (1+log(TF(t,d))) log(N/df(t))
Best Case: Documents with high term repetition
Worst Case: Short documents with few terms
Speed: Fast (O(nm))
Recommended For: Long documents, reviews, articles with repetitive language
Tip: Better than standard TF-IDF for documents with varying term densities.
Boolean Search

Binary matching - documents either match or don't.

How It Works: Checks if all query terms exist in document (AND logic)
Formula:
Boolean(D,Q) = 1 if all terms in Q exist in D, else 0
Best Case: Exact requirements, filtering
Worst Case: Ranking needed, partial matches
Speed: Very Fast (O(n))
Recommended For: Exact matching, database-style queries, filtering
Warning: No ranking! All results have equal importance.
Quick Comparison

BM25 Excellent Fast Medium General searches

Performance Tips
For Speed:
  • Use Boolean for simple filtering
  • Use BM25 for fast ranked results
  • Avoid Vector Space with 1000+ documents
For Accuracy:
  • Use BM25 for most cases
  • Use Log TF-IDF for long documents
  • Use Vector Space for similarity
For Long Documents:
  • BM25 - Best overall
  • Log TF-IDF - Good alternative
For Short Queries:
  • TF-IDF - Simple and effective
  • Boolean - Exact matching

What Each Metric Means
  • Precision: % of results that are relevant
  • Recall: % of all relevant documents found
  • F1 Score: Balance between precision and recall
  • Accuracy: Overall correctness of the results
